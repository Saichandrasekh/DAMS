# DAMS - Digital Attendance Management System
## System Architecture & Technical Manual

### 📝 Overview
DAMS is a high-performance, multi-tenant Attendance Management System built with **Flask** and **SQLite**. It is designed to scale to 100+ schools while running on local hardware via a professional production stack.

---

### 🏗️ Technical Stack
- **Backend**: Python 3.x + Flask
- **Database**: SQLite (optimized with multi-tenant indexing)
- **Production Server**: Waitress (handles concurrency on Windows)
- **Reverse Proxy**: Nginx (serves static assets and provides SSL/Caching)
- **Connectivity**: Cloudflare Tunnel (expose local server to global URL)

---

### 🔐 Database Design (Multi-Tenancy)
Every table includes a `school_id` column. **Important Rules:**
1. **Scoping**: All queries MUST include `WHERE school_id = ?` to prevent data leakage between schools.
2. **Indexing**: Primary indexes are placed on `(school_id, email)` and `(school_id, role)` for fast lookups.
3. **Deletion**: We use a **Two-Stage Deletion** pattern:
   - **Active Record** + Delete -> `is_active = 0` (Archive)
   - **Inactive Record** + Delete -> `DELETE` (Permanent)

### 🎨 Design System (Premium UI)
The project uses a modern aesthetic defined in `static/css/style.css`:
- **Typography**: Outfit (Headers) & Inter (Body).
- **Colors**: HSL variables (Indigo Primary).
- **Effects**: Glassmorphism (sidebars), dynamic shadows, and 400ms slide-up animations.
- **Popups**: Global SweetAlert2 integration in `main.js` via `data-confirm` attributes.

### 🚀 Production Workflow
To launch the system professionally:
1. **Launch App**: `python run_prod.py` (Starts Waitress on port 5000).
2. **Nginx**: Configuration at `C:\nginx-x.x\conf\nginx.conf` proxies `attend.kautech.co.in` to port 5000.
3. **Static Files**: Nginx serves `/static/` directly from `d:\attendence\backend\static\` for maximum speed.
